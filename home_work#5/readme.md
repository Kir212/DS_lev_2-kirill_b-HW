# Реализация DCGAN для генерации изображений на датасете Fashion-MNIST

Реализация глубокой генеративно-состязательной нейросети (Deep Convolutional GAN) на фреймворке PyTorch. Проект разработан и оптимизирован для запуска в среде Google Colab с использованием аппаратного ускорения GPU (T4).

В качестве обучающей выборки используется датасет Fashion-MNIST (изображения элементов одежды, обуви и аксессуаров).

---

## Архитектура моделей

### 1. Генератор (Generator)
Принимает на вход вектор случайного шума (размерность latent_dim = 100) и разворачивает его в полноценное одноканальное изображение размером 28x28 пикселей.
* Полносвязный слой (Linear) для первичного проецирования вектора.
* Слои транспонированной свертки (ConvTranspose2d) для апсэмплинга (увеличения пространственного разрешения).
* Пакетная нормализация (BatchNorm2d) для стабилизации обучения.
* Функции активации: ReLU в скрытых слоях и Tanh на выходе (перевод пикселей в диапазон [-1, 1]).

### 2. Дискриминатор (Discriminator)
Классический сверточный бинарный классификатор. Принимает изображение (28x28) и определяет вероятность того, является ли оно настоящим (1) или сгенерированным (0).
* Сверточные слои (Conv2d) для извлечения высокоуровневых признаков с уменьшением размерности.
* Функции активации: LeakyReLU(0.2) во всех скрытых слоях для предотвращения проблемы затухающих градиентов.
* Финальный слой: Flatten -> Linear -> Sigmoid для получения вероятностной оценки.

---

## Параметры обучения (Hyperparameters)

* Размер батча (Batch Size): 128
* Количество эпох (Epochs): 15 (7020 шагов оптимизации)
* Размерность латентного пространства: 100
* Функция потерь (Loss): Бинарная кросс-энтропия (BCELoss)
* Оптимизатор: Adam для обеих сетей
  * Learning Rate = 0.0002
  * Betas = (0.5, 0.999)

---

## Метрики и результаты состязания

В процессе обучения было достигнуто динамическое Равновесие Нэша, при котором сети развивались синхронно, не подавляя друг друга.

### Итоговая статистика обучения:
* Дискриминатор (Loss D): Стартовое: 1.0745, Финальное: 0.6110 (Среднее: 0.6787)
* Генератор (Loss G): Стартовое: 1.5122, Финальное: 1.9054 (Среднее: 1.5701)

Стабильное удержание ошибки Дискриминатора в диапазоне 0.5–0.7 к концу 15 эпохи подтверждает, что Генератор научился обманывать классификатор примерно в 50% случаев, создавая реалистичные текстуры одежды. Коллапса моды (Mode Collapse) зафиксировано не было.

---

## Выводы и анализ результатов

По итогам 7020 итераций обучения были сделаны следующие выводы:

1. Динамика состязания сетей. Снижение ошибки Дискриминатора со стартовых 1.0745 до финальных 0.6110 и стабилизация среднего значения на уровне 0.6787 свидетельствуют о корректном обучении классификатора. Дискриминатор развивался достаточно быстро, чтобы выступать строгим критиком, но не подавил Генератор.

2. Поведение Генератора. Рост ошибки Генератора с 1.5122 до 1.9054 является нормальным для состязательных моделей. По мере того как Дискриминатор учился замечать мелкие дефекты, Генератору становилось сложнее его обманывать, что заставило сеть перейти от генерации грубых силуэтов к проработке реалистичных текстур и деталей одежды.

3. Стабильность системы. Показатели стандартного отклонения (0.1815 для Дискриминатора и 0.3248 для Генератора) подтверждают отсутствие критических скачков, взрыва градиентов или коллапса моды. Начиная с 3500-й итерации, система вышла на плато динамического равновесия Нэша, при котором обе сети сбалансировали свои силы. Математические параметры оптимизатора Adam подобраны успешно.
