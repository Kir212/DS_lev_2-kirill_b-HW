# Telegram-бот ТВ-Консультант на PyTorch

Реализация контекстного чат-бота для подбора и заказа телевизоров в Telegram. Проект выполнен на базе архитектуры Feed-Forward нейронной сети с использованием библиотеки PyTorch и развернут в среде Jupyter Notebook / Google Colab.

Проект разработан на основе базового шаблона patrickloeber/pytorch-chatbot, но существенно доработан под бизнес-логику, снабжен системой сквозного логирования и гибридным контекстным процессором.

---

## Ключевые особенности и доработки

- **Обучаемая нейросеть**: Классический Feed-Forward NN (многослойный перцептрон) для классификации намерения (интентов) пользователя.
- **Динамический инпут**: Размер входного слоя автоматически подстраивается под актуальный размер словаря (all_words) при изменении файла интентов, исключая ошибки несоответствия матриц (RuntimeError: mat1 and mat2 shapes cannot be multiplied).
- **Повышенный порог уверенности**: Минимальный порог уверенности модели поднят до 0.70, что позволяет эффективно отсекать ложные срабатывания нейросети на абстрактный текстовый шум и переводить диалог на вежливую универсальную заглушку.
- **Гибридный движок ответов (Нейросеть + Regex)**: Короткие фразы (Да, LG, 55), специфические типы комнат (спальня, гостиная), а также сложные качественные запросы (большой, умный) перехватываются регулярными выражениями до отправки в сеть.
- **Защита от числовых коллизий**: Регулярные выражения жестко ограничены диапазоном реальных диагоналей (от 22 до 99 дюймов). Это исключает баг, при котором ввод клиентом бюджета (например, 1000) распознавался как запрос на телевизор в 000 дюймов.
- **Память контекста (Contextual Memory)**: Бот пошагово накапливает параметры пользователя (диагональ и бренд) в структуре user_data. Как только оба ключевых параметра собраны, скрипт автоматически переводит диалог в стадию оформления, предотвращая зацикливание.
- **Лидогенерация**: После успешного подбора телевизора бот меняет внутренний статус сессии и запрашивает имя и номер телефона клиента для последующей передачи заказа менеджеру.
- **Сквозное логирование**:
  - training.log — логирование процесса обучения (эпохи, текущие финальные потери Loss).
  - dialogs.log — история переписки (ID пользователя, текст запроса, ответ бота, сработавший тег и уверенность модели Conf).

## Архитектура нейронной сети

Для классификации текстовых намерений используется полносвязная нейронная сеть (Feed-Forward Neural Network) со следующей структурой:
1. **Входной слой (Linear)**: Размерность динамически равна общему количеству уникальных токенизированных слов (input_size).
2. **Скрытый слой (Linear + ReLU)**: Размерность hidden_size = 8 для выявления скрытых связей в мешке слов.
3. **Выходной слой (Linear)**: Размерность равна количеству уникальных тегов/классов (output_size).
4. **Оптимизация**: Функция потерь CrossEntropyLoss и оптимизатор Adam (learning rate = 0.001).

Предобработка текста включает токенизацию средствами nltk, стемминг (выделение основы слова) и очистку от знаков препинания с последующей векторизацией по методу Мешка слов (Bag of Words).

## Установка и запуск

### 1. Подготовка окружения (для локального запуска)
Если вы запускаете проект не в Google Colab, установите необходимые зависимости:

```bash
pip install torch numpy python-telegram-bot nltk
```

Для корректной работы токенизатора NLTK на русском языке, скрипт автоматически подгружает необходимый пакет данных:
```python
import nltk
nltk.download('punkt_tab')
```

### 2. Подготовка базы знаний (intents.json)
Создайте или отредактируйте файл intents.json в корневой директории. Пример структуры:
```json
{
  "intents": [
    {
      "tag": "greeting",
      "patterns": ["Привет", "Здравствуйте", "Добрый день"],
      "responses": ["Приветствую! Я помогу вам подобрать идеальный телевизор. Какая диагональ вас интересует?"]
    },
    {
      "tag": "tv_selection",
      "patterns": ["Посоветуй ТВ", "Какой бренд выбрать", "Что купить"],
      "responses": ["Среди брендов сейчас лидируют Samsung, LG (OLED матрицы) и Xiaomi/TCL (лучшее соотношение цена/качество). Какой у вас бюджет?"]
    }
  ]
}
```

### 3. Обучение модели
Запустите блок обучения. Прогресс будет отображаться в консоли и записываться в training.log:
```text
INFO: Начало обучения ТВ-консультанта...
INFO: Эпоха [100/1000], Ошибка (Loss): 1.1245
INFO: Эпоха [500/1000], Ошибка (Loss): 0.2314
INFO: Обучение завершено. Финальная ошибка: 0.0102
INFO: Файл модели успешно сохранен в data.pth
```

### 4. Запуск Telegram-бота
1. Создайте бота через @BotFather в Telegram и получите API Токен.
2. Вставьте ваш токен в переменную TOKEN внутри асинхронной функции запуска бота.
3. Запустите ячейку. Бот перейдет в режим асинхронного пуллинга (ожидания сообщений).

## Краткие выводы по проекту

   1. Успешная интеграция: Разработан и развернут в Telegram полноценный контекстный чат-бот для подбора телевизоров на базе полносвязной нейронной сети (Feed-Forward NN) с использованием PyTorch.
   2. Гибридная архитектура: Сочетание классического машинного обучения (Bag of Words) и регулярных выражений (Regex) позволило решить проблему ложных срабатываний нейросети на коротких, абстрактных или зашумленных фразах («Да», «LG», «Большой и умный»).
   3. Управление контекстом: Реализована система сохранения состояния (user_data), которая пошагово накапливает параметры подбора (бренд и диагональ), предотвращает зацикливание диалога и успешно доводит пользователя до этапа лидогенерации (сбора контактных данных).
   4. Отказоустойчивость: Устранены критические ошибки несовпадения размерностей матриц слоев за счет динамического вычисления размера входного слоя на основе актуальной длины словаря интентов.
   5. Прозрачность и контроль: Сквозное логирование процесса обучения (training.log) и истории переписки (dialogs.log) обеспечивает полную прозрачность работы модели и позволяет оперативно собирать данные для улучшения базы знаний.
